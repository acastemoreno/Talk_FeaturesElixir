Pattern Matching

2 partes:

  • Data: Expresion evaluada contra la cual se hara una comparación.
  • Patron: Forma de data. -Contiene variables.

Pattern Matching

Elixir tratará de asignar valores a las variables para que la comparación sea verdadera.

  • La asignación de variables es una posible consecuencia de pattern matching.

Ejemplos

#Asignación?
iex> nombre = "Albert"

#Destructuring?
iex> {nombre, _apellido} = {"Albert", "Castellano"}
iex> {value, value} = {28, 27}
iex> {^nombre, apellido} = {"Jose", "Valim"}

#Pattern Matching!
iex> [a,_,_,_, %{value: a}] = [1, 2, 3, 4, %{value: 1}]
iex> [a,_,_,_, %{value: a}] = [1, 2, 3, 4, %{value: 2}]

Funciones

Divididas en dos tipos.

Funciones anonimas

Funciones anonimas

#fn(<parametro1>, <parametro2>, ....) -> <ejecucion y retorno> end

iex> velocidad = fn (altura) -> :math.sqrt(2 * 9.81 * altura) end
#Function<6.52032458/1 in :erl_eval.expr/5>
iex> velocidad.(3)
7.672027111526653

iex> sum = fn (a, b) -> a + b end
#Function<12.17052888 in :erl_eval.expr/5>
iex> sum.(1, 2)
3

Funciones anonimas y pattern matching

iex> velocidad = fn
...> (altura) when is_number(altura) -> :math.sqrt(2 * 9.81 * altura)
...> (_) -> "Qué paso amiguito?"
...> end
#Function<6.52032458/1 in :erl_eval.expr/5>

iex> velocidad(3)
7.672027111526653

iex> velocidad.("Holi")
"Qué paso amiguito?"

Shortcut &

iex> cociente_residuo = &{ div(&1,&2), rem(&1,&2) }
iex> cociente_residuo.(13, 5)
{2, 3}
iex> lista = &[&1, &1 * 2, &1 * 3]
iex> lista.(5)
[5, 10, 15]

Funciones con nombre

Funciones con nombre

#file: caida.exs
defmodule Caida do
  def velocidad(altura) do
    :math.sqrt(2 * 9.81 * altura)
  end
end

Funciones con nombre y pattern matching

#file: factorial.exs
defmodule Factorial do
  def de(0), do: 1
  def de(n) when is_number(n), do: n * de(n-1)
  def de(_), do: "Que paso amiguito?"
end
